## 1. 产品概述

### 1.1 游戏定位

一款以 **2026 年世界杯** 为背景的网页端即时策略小游戏。玩家扮演球队经理，通过 **选教练 → 定阵型 → 抽球员 → 应对比赛突发事件** 的完整流程，组建一支跨国家混编的 26 人大名单球队，征战完整的世界杯赛程，最终根据成绩获得排名。

### 1.2 核心玩法循环

```
创建球队 → 选主教练 → 选阵型 → 逐位置抽选球员(26人) → 小组赛(3场) → 淘汰赛 → 成绩结算 → 分享
```

### 1.3 与虎扑世界杯小游戏的核心差异

| 维度 | 虎扑小游戏 | 本游戏 |
|------|----------|--------|
| 球队构成 | 选一个国家队 | 跨国家混编，每位置随机抽国再选人 |
| 教练选择 | 固定或无 | 48 位真实教练，互相克制，各自优缺点 |
| 阵型 | 固定 | 教练确定后选阵型，变阵有风险/回报 |
| 球员选人 | 预设大名单 | 每位置随机国家→挑球员，可重随 2 次 |
| 突发事件 | 简单或无 | 三级事件系统，50+ 事件池，影响比赛 |
| 评分 | 模糊 | 100 分制 + 进攻/防守/中场三围 |
| 社交 | 截图 | 精美成绩卡片 + 双人同设备 PK |

### 1.4 目标平台

- **桌面端:** Chrome / Edge / Firefox（主要开发目标）
- **移动端:** 响应式适配，手机浏览器可玩，扫码即玩

## 4. 教练系统

### 4.1 打法风格（5 种 + 克制环）

```
       高位压迫 ──克──→ 传控渗透
          ▲               │
          │               ▼
       铁桶大巴         防守反击
          │               │
          ▼               ▼
       两翼齐飞 ←──克── 传控渗透

完整克制链:
  高位压迫 → 克 传控渗透、两翼齐飞
  传控渗透 → 克 防守反击、铁桶大巴
  防守反击 → 克 高位压迫、两翼齐飞
  两翼齐飞 → 克 铁桶大巴、防守反击
  铁桶大巴 → 克 高位压迫、两翼齐飞
```

| 风格 | 特点 | 体能消耗 | 克制 | 被克制 |
|------|------|---------|------|--------|
| 🔴 高位压迫 | 前场逼抢凶猛、转换快 | 高 | 传控、两翼齐飞 | 防反、铁桶 |
| 🔵 传控渗透 | 高控球率、短传渗透 | 中 | 防反、铁桶 | 高位压迫、两翼齐飞 |
| 🟢 防守反击 | 放弃控球、抓身后空当 | 低 | 高位压迫、两翼齐飞 | 传控、铁桶 |
| 🟡 两翼齐飞 | 边路突破、传中抢点 | 中高 | 铁桶、防反 | 高位压迫、传控 |
| 🟣 铁桶大巴 | 极致防守、1-0 主义 | 低 | 高位压迫、两翼齐飞 | 传控、防反 |

### 4.2 教练属性

每位教练 3 个属性（满分 20）：

- **进攻加成 (ATT):** 叠加到球队进攻综合评分
- **防守加成 (DEF):** 叠加到球队防守综合评分
- **临场应变 (ADP):** 影响突发事件选项效果 + 变陌生阵型时的惩罚减免

### 4.3 教练数据结构

```json
{
  "id": "coach-001",
  "name": "德尚",
  "nameEn": "Didier Deschamps",
  "nationality": "法国",
  "avatar": "/images/coaches/deschamps.png",
  "style": "防守反击",
  "att": 12,
  "def": 18,
  "adp": 15,
  "preferredFormations": ["4-2-3-1", "4-3-3", "4-4-2"],
  "description": "实用主义大师，擅长大赛淘汰赛博弈"
}
```

### 4.4 教练克制加成

- 教练打法克制对方 → 球队综合分 **+8%**
- 被对方克制 → **-8%**
- 相同风格 → 无影响

## 5. 阵型系统

### 5.1 可选阵型（8 种）

| 阵型 | 位置分布 | 特点 |
|------|---------|------|
| **4-3-3** | GK / LB·CB·CB·RB / CM·CM·CM / LW·ST·RW | 边路强、中场三人控制力好 |
| **4-2-3-1** | GK / LB·CB·CB·RB / CDM·CDM / CAM·LW·RW / ST | 前腰核心、攻守最均衡 |
| **4-4-2** | GK / LB·CB·CB·RB / LM·CM·CM·RM / ST·ST | 双前锋、经典英式打法 |
| **3-5-2** | GK / CB·CB·CB / CDM·CM·CM·LM·RM / ST·ST | 中场人数优势、翼卫关键 |
| **5-3-2** | GK / LWB·CB·CB·CB·RWB / CM·CM·CM / ST·ST | 铁桶防守、翼卫反击 |
| **4-1-4-1** | GK / LB·CB·CB·RB / CDM / LM·CM·CM·RM / ST | 单后腰保护、单箭头 |
| **3-4-3** | GK / CB·CB·CB / CM·CM·LM·RM / LW·ST·RW | 全攻全守、三前锋 |
| **4-3-2-1** | GK / LB·CB·CB·RB / CM·CM·CM / CAM·CAM / ST | 圣诞树、中路极度密集 |

### 5.2 阵型克制关系

| 阵型 | 克制 | 被克制 |
|------|------|--------|
| 4-3-3 | 4-4-2（边路 3v2） | 4-2-3-1、5-3-2 |
| 4-2-3-1 | 4-3-3、4-4-2 | 3-5-2、4-1-4-1 |
| 4-4-2 | 3-5-2、5-3-2 | 4-3-3、4-2-3-1 |
| 3-5-2 | 4-2-3-1、4-4-2 | 4-3-3、3-4-3 |
| 5-3-2 | 4-3-3、3-4-3 | 4-4-2、4-1-4-1 |
| 4-1-4-1 | 4-2-3-1、3-5-2 | 4-4-2、3-4-3 |
| 3-4-3 | 3-5-2、4-1-4-1 | 5-3-2、4-3-3 |
| 4-3-2-1 | 4-2-3-1 | 4-3-3、4-4-2 |

克制时 +5% 综合分，不克制视为均势。

### 5.3 教练阵型熟练度

- 每位教练 **2-3 个擅长阵型**（熟练度 100%），选自擅长阵型 **无惩罚**
- 比赛中变阵到非擅长阵型：
  - 基础惩罚：**-3% ~ -8%** 综合分
  - 教练 `ADP ≥ 15`：惩罚减半或免疫
  - 但若新阵型克制对方当前阵型：获得 +5% 加成，可能抵消甚至超过惩罚

| 场景 | ADP < 15 | ADP ≥ 15 |
|------|----------|----------|
| 变熟悉阵 + 不克制 | 0% | 0% |
| 变熟悉阵 + 克制对方 | +5% | +5% |
| 变陌生阵 + 不克制 | -5% | -2% |
| 变陌生阵 + 克制对方 | -3% + 5% = +2% | -1% + 5% = +4% |

## 6. 球员系统与选人流程

### 6.1 选人完整流程

```
选教练 → 选阵型 → 按阵型位置逐位选首发(11人) → 选替补(15人) → 阵容总览
```

#### 逐位置操作流程（每个位置重复）

1. 阵型图上显示 **⊕ 加号** 在待选位置
2. 点击加号 → 弹出选人窗口
3. 系统随机抽取一个国家（显示国旗 + 国名动画）
4. 展示该国该位置所有入选世界杯的球员列表（头像、姓名、评分、擅长位置标签）
5. 玩家点选 1 人加入阵容
6. 不满意可点 **"重新随机"**（最多 **2 次**），换另一个国家
7. 确认后球员填入阵型图，进入下一位置

#### 选人顺序

```
门将(GK) → 后卫(按阵型从左到右) → 中场(按阵型从后到前) → 前锋(按阵型从左到右)
```

### 6.2 球员属性

```json
{
  "id": "player-001",
  "name": "姆巴佩",
  "nameEn": "Kylian Mbappé",
  "nationality": "法国",
  "avatar": "/images/players/mbappe.png",
  "rating": 91,
  "positions": ["ST", "LW", "RW"],
  "stats": {
    "speed": 97,
    "shooting": 88,
    "passing": 80,
    "defending": 36,
    "physical": 78
  }
}
```

| 属性 | 说明 |
|------|------|
| 头像 | 真实照片 |
| 姓名 | 中文名 + 英文名 |
| 综合评分 | 0-100 |
| 擅长位置 | 缩写标签: `GK` `CB` `LB/RB` `LWB/RWB` `CDM` `CM` `CAM` `LM/RM` `LW/RW` `ST` |
| 关键能力 | 速度/射门/传球/防守/体能，各 0-100 |

### 6.3 去重规则

- 已选球员的国家不会在后续轮次的随机池中出现
- 即：**不能从同一个国家抽两次**
- 48 个国家 → 26 人大名单，去重池充裕，无需担心不够用

### 6.4 替补选择（15 人）

- 首发 11 人选完后进入替补选人页
- 不限制位置，自由抽取（同样每国最多出现一次）
- 系统提示推荐覆盖的位置（如还缺后卫、还缺门将替补等）
- 选满 15 人自动跳转阵容总览

### 6.5 球队综合评分计算

```
进攻评分 = 前锋群评分均值 × 0.5 + 中场攻击属性 × 0.3 + 边卫攻击 × 0.2 + 教练.ATT
防守评分 = 后卫群评分均值 × 0.5 + 后腰评分 × 0.25 + 门将评分 × 0.25 + 教练.DEF
中场评分 = 中场群评分均值 × 0.6 + 前腰/后腰 × 0.4

综合实力 = 进攻 × 0.35 + 防守 × 0.35 + 中场 × 0.30
```

三围数据用**雷达图**展示在阵容总览页。

## 7. 比赛系统

### 7.1 赛制（2026 世界杯真实赛制）

- **48 支球队**，12 组 × 4 队（A-L 组）
- **小组赛:** 每队踢 3 场，胜 3 分 / 平 1 分 / 负 0 分
  - 每组 **前 2 名**（24 队）+ **8 个成绩最好的小组第 3** = 共 **32 队**晋级
  - 小组第 3 排名规则：积分 → 净胜球 → 进球数 → 公平竞赛积分 → 抽签
  - 小组内同分排序：**相互战绩优先**（积分→净胜球→进球），再比总净胜球→总进球→公平竞赛分→抽签
- **淘汰赛（新增 32 强赛）:** 32 强 → 16 强 → 8 强 → 4 强 → 季军赛 → 决赛
  - 冠军共踢 **8 场**比赛（小组 3 场 + 淘汰赛 5 场）
  - 淘汰赛平局 → 加时赛 30 分钟（上下半场各 15 分钟）→ 点球大战
- **总比赛场次:** 104 场（小组赛 72 场 + 淘汰赛 32 场）

### 7.2 比赛判定公式

```
球员实力分(P) = Σ(11 个首发评分 × 阵型位置权重) / 11
战术匹配分(T) = 教练打法克制? ± 8% + 阵型克制? ± 5% + 球员位置匹配度
事件决策分(E) = Σ(每事件选项得分)，区间约 -10 ~ +15
随机波动     = ±5%（模拟运气成分）
体能扣减     = 连续作战消耗 0 ~ -5 分

己方综合分 = P × 0.55 + T×100 × 0.25 + E × 0.15 + 随机波动 - 体能扣减
对方综合分 = 同上（AI 队 P 为预设纸面实力分）

分差 > 8   → 大胜 (如 3-0, 4-1)
分差 2 ~ 8 → 小胜 (如 1-0, 2-1)
分差 < 2   → 平局 (0-0, 1-1)
分差 -2~-8 → 小负 (0-1, 1-2)
分差 < -8  → 大负 (0-3, 1-4)
```

### 7.3 AI 球队模拟

- AI 队数据简化：给出**国家名 + 国旗 + 纸面实力分（三围）+ 分档**（共 48 队，分 4 档）
- 12 个小组中，玩家球队占 1 组，其余 11 组纯 AI 模拟
- AI vs AI 模拟使用**加权概率**判定，确保结果符合现实：

| 对阵 | 强队胜率 | 平局率 | 弱队胜率 |
|------|---------|--------|---------|
| 一档 vs 四档 | 85% | 12% | 3% |
| 一档 vs 三档 | 75% | 18% | 7% |
| 一档 vs 二档 | 62% | 25% | 13% |
| 同档对抗 | 40% | 30% | 30% |

- 四强/决赛阶段基本只有一档 + 少量二档球队，符合现实格局
- 弱队连续爆冷概率极低（约 3%），但保留"黑马剧本"可能性

### 7.4 体能系统

- 每场比赛首发球员体能消耗取决于打法风格：
  - 高位压迫：消耗 **大**
  - 传控渗透：消耗 **中**
  - 防守反击：消耗 **小**
- 两场间隔短时体能恢复不足，下一场会影响球员评分（-1 ~ -5 分）
- 淘汰赛前系统提示玩家考虑轮换
- 替补球员上场不受体能惩罚

### 7.5 比赛呈现

- **文字直播流**：模拟比赛进程的文字滚动动画
- **比分实时更新**：顶部大比分牌
- **事件弹窗**：中断文字流，玩家做决策后继续
- **赛后统计**：控球率、射门数、传球成功率等根据评分反向生成

## 8. 突发事件系统

### 8.1 事件分级

| 级别 | 触发 | 交互 | 频率 |
|------|------|------|------|
| 🟢 普通 | 自动 | 消息栏滚动 | 每场 5-8 次 |
| 🟡 重要 | 弹出二选一 | 玩家 15 秒内选 | 每场 2-4 次 |
| 🔴 重大 | 弹出三选一 | 玩家 20 秒内选 | 每场 1 次 |

### 8.2 事件影响

| 选择质量 | 分数波动 |
|----------|---------|
| 最佳决策 | +3 ~ +5 |
| 可行决策 | +1 ~ +2 |
| 糟糕决策 | -2 ~ -5 |

教练 **临场应变 (ADP)** 属性放大优势选择的收益：
- ADP 高 → 最佳决策加分更多（最高 +6）
- ADP 低 → 糟糕决策扣分更多（最高 -6）

### 8.3 事件池（50+ 条，部分示例）

#### 伤病类
| 事件 | 重要度 | 选项 |
|------|--------|------|
| 核心球员被铲倒，表情痛苦 | 🔴 重大 | A.换下休息保护 / B.打封闭坚持 / C.观察 5 分钟再决定 |

## 9. 双人 PK 模式

### 9.1 模式说明

**同设备轮换 PK**，流程：

1. 首页点击"双人 PK"
2. 玩家 A 走完整流程：选教练 → 阵型 → 球员 → 踢世界杯 → 出成绩
3. 系统保存 A 的成绩数据
4. 屏幕提示"交给玩家 B"
5. 玩家 B 走完相同流程
6. 系统自动对比双方成绩，弹出 PK 对比页

### 9.2 胜负判定

| 优先级 | 判定标准 |
|--------|---------|
| 1 | 谁走得更远（冠军 > 亚军 > 季军 > 4强 > 8强 > 16强 > 32强 > 小组赛） |
| 2 | 同轮次比净胜球 |
| 3 | 同净胜球比总进球 |
| 4 | 完全相同则平手 |

## 11. 数据需求

### 11.1 数据文件清单

| 文件 | 内容 | 规模 |
|------|------|------|
| `teams.json` | 48 支参赛队（国旗、国名、纸面实力分档、三围） | 48 条 |

| `events.json` | 突发事件（类型、选项、分数影响、触发条件） | 50+ 条 |
| `ai-teams.json` | AI 球队模板（各国家队纸面阵容三围预设） | 48 条 |

### 11.2 数据搜集策略

- 2026 世界杯 48 支晋级队：以 FIFA 官方已公布的分组为准（12 组 × 4 队）
- 球员评分：参考 FIFA/EA Sports FC 最新评分 + 近期表现
- 教练信息：维基百科 + FIFA 官方资料
- 数据搜集工作将在设计确认后展开，48 队数据量较大（~1,248 名球员），利用 AI 批量搜集

## 12. UI/UX 设计方向

### 12.1 视觉风格

- **参考虎扑 / 懂球帝**：深色主题为主（护眼 + 高级感），关键信息高亮
- **配色**：深绿(#0D2818) + 金色(#D4A843) + 白(#FFFFFF)
- **字体**：中文用系统默认（微软雅黑/PingFang），数字用等宽字体
- **卡片式布局**：每个信息模块独立卡片，圆角 + 微阴影

### 12.2 关键页面参考

- **选教练页**：类似球员卡收集，48 张教练卡片网格排列，可筛选打法风格
- **选阵型页**：足球阵型图（圆点代表球员位置），点击高亮选中
- **选球员页**：阵型图主体 + 底部弹出面板（国家队 + 球员列表）
- **比赛页**：上方比分牌 + 中间文字直播 + 突发事件弹窗
- **成绩卡片**：精美单页排版，适合截图分享

### 12.3 响应式设计

- 桌面端：最大宽度 480px 居中（手机优先设计，桌面也好看）
- 移动端：全屏适配，触摸友好
- 阵型图、雷达图等 SVG 元素等比缩放
